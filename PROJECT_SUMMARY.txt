PROJECT SUMMARY - LUMINA (SMART HOME ENERGY PREDICTION & MONITORING)
=====================================================================

1) WHAT THE PROJECT IS
----------------------
Lumina is a Streamlit-based prototype for household energy analytics.
It combines:
- Historical consumption analysis
- AI-based 24-hour energy forecasting
- Live virtual smart meter simulation
- Weather-aware demand context
- Anomaly alerts
- Solar generation and ROI estimation
- CSV export and quick demo mode for presentations

Main app entry point:
- app.py

Core modules:
- src/inference_ceew.py      -> Production model inference
- src/preprocessing_ceew.py  -> CEEW preprocessing for app usage
- src/weather.py             -> Live/forecast weather integration
- src/virtual_meter.py       -> Virtual smart meter telemetry
- src/anomaly.py             -> Usage anomaly detection
- src/solar.py               -> Solar potential and ROI logic
- src/chatbot.py             -> Basic assistant responses


2) DATASETS USED
----------------
Primary active dataset (current app + production model):
- enhanced_datasets/CEEW_BR04_with_weather.csv

Dataset origin/context in this project:
- archive_extracted/CEEW - Smart meter data Bareilly 2020.csv (meter BR04 source)
- datasets_extracted/dataset/WeatherData.csv (weather context source for augmentation)

What the active dataset contains:
- Smart meter timestamped consumption and electrical signals
- Weather attributes mapped to timestamps
- Used after hourly resampling in preprocessing/inference flow

4) FEATURES ADDED / USED FOR FORECASTING
-----------------------------------------
The active production model uses 17 engineered features:

Lag & rolling features:
1. lag_1h
2. lag_24h
3. lag_168h
4. rolling_mean_24h
5. rolling_std_24h

Time features:
6. hour
7. day_of_week
8. month
9. hour_sin
10. hour_cos

Electrical features:
11. z_Avg Voltage (Volt)
12. z_Avg Current (Amp)
13. y_Freq (Hz)

Weather features:
14. temp
15. humidity
16. pressure
17. wind_speed
